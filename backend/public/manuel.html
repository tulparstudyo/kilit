📖 Kurulum Kılavuzu

Tulpar Kilit sistemini kurmak için aşağıdaki iki aşamayı takip edin. Sadece masaüstü uygulamasını kuracaksanız 1. aşama yeterlidir. Kendi sunucunuzda barındırmak istiyorsanız 2. aşamayı da tamamlayın.

Bu aşama Pardus Linux (veya Debian tabanlı) bilgisayarlara masaüstü kilit uygulamasını kurar. Varsayılan olarak kilit.dosyamosya.com sunucusuna bağlanır.
1 Gereksinimler

Aşağıdaki bileşenlerin sisteminizde kurulu olması gerekir:

  • Pardus Linux veya Debian/Ubuntu tabanlı dağıtım
  • Python 3.8 veya üzeri
  • İnternet bağlantısı (ilk kurulum için)
  • Sudo yetkisi
2 Otomatik Kurulum (Önerilen)

Tek komutla tüm kurulumu gerçekleştirin:

# Kurulum scriptini indir ve çalıştır
wget -qO- https://raw.githubusercontent.com/tulparstudyo/kilit/main/kurulum.sh | bash

Script şunları otomatik yapar:

  • Sistem bağımlılıklarını kurar (GTK3, PyGObject, pip)
  • Projeyi /opt/tulpar-kilit dizinine indirir
  • Python bağımlılıklarını yükler
  • Yapılandırma ayarlarını sorar (kurum kodu, API adresi vb.)
  • Tüm kullanıcılar için otomatik başlatma ve masaüstü kısayolu oluşturur
3 Manuel Kurulum (Alternatif)

Otomatik kurulum yerine adım adım yapmak isterseniz:

# 1. Sistem paketlerini kur
sudo apt-get update
sudo apt-get install -y python3-gi python3-gi-cairo gir1.2-gtk-3.0 python3-pip git

# 2. Projeyi indir
sudo git clone https://github.com/tulparstudyo/kilit.git /opt/tulpar-kilit

# 3. Python bağımlılıklarını kur
pip3 install --break-system-packages -r /opt/tulpar-kilit/desktop/requirements.txt

# 4. Uygulamayı test et
python3 /opt/tulpar-kilit/desktop/tulpar_lock.py
4 Yapılandırma

Otomatik kurulum sırasında bu bilgiler sorulur. Manuel kurulumda /opt/tulpar-kilit/desktop/.env dosyasını düzenleyin:

API_URL=https://kilit.dosyamosya.com
INSTITUTION_CODE=123456
INSTITUTION_NAME=Örnek Okulu
OFFLINE_SECRET=tulpar_offline_secret_key
UNLOCK_DURATION=40
DeğişkenAçıklamaVarsayılan
API_URLBackend sunucu adresihttps://kilit.dosyamosya.com
INSTITUTION_CODEKurum kodu (yöneticinizden alın)—
INSTITUTION_NAMEKilit ekranında görünecek kurum adı—
OFFLINE_SECRETÇevrimdışı kilit açma anahtarı—
UNLOCK_DURATIONKilitsiz kalma süresi (dakika)40
⚠️ INSTITUTION_CODE ve OFFLINE_SECRET değerlerini kurum yöneticinizden temin edin.
5 Güncelleme

Kurulum scriptini tekrar çalıştırmak mevcut kurulumu günceller. Yapılandırma dosyanız korunur:

wget -qO- https://raw.githubusercontent.com/tulparstudyo/kilit/main/kurulum.sh | bash
✅ Kurulum tamamlandığında uygulama her oturum açılışında otomatik başlar ve masaüstünde kısayol oluşturulur.
Bu aşama kendi sunucunuzda Tulpar Kilit backend'ini barındırmak isteyenler içindir. Varsayılan sunucuyu (kilit.dosyamosya.com) kullanacaksanız bu aşamayı atlayabilirsiniz.
1 Sunucu Gereksinimleri
  • Ubuntu 20.04+ veya Debian 11+ sunucu
  • Node.js 16 veya üzeri
  • MySQL 8.0 veya MariaDB 10.5+
  • Minimum 1 GB RAM, 10 GB disk
  • Geçerli bir alan adı ve SSL sertifikası (HTTPS zorunlu)
2 Sistem Hazırlığı 
# Node.js kur (NodeSource üzerinden)
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs

# MySQL kur
sudo apt-get install -y mysql-server

# MySQL güvenlik ayarları
sudo mysql_secure_installation

# Nginx kur (reverse proxy için)
sudo apt-get install -y nginx certbot python3-certbot-nginx
3 Veritabanı Oluşturma 
# MySQL'e bağlan
sudo mysql -u root -p

# Veritabanı ve kullanıcı oluştur
CREATE DATABASE tulpar_kilit CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'tulpar'@'localhost' IDENTIFIED BY 'GÜÇLÜ_BİR_ŞİFRE';
GRANT ALL PRIVILEGES ON tulpar_kilit.* TO 'tulpar'@'localhost';
FLUSH PRIVILEGES;
EXIT;
⚠️ GÜÇLÜ_BİR_ŞİFRE yerine rastgele, uzun bir şifre belirleyin.
4 Proje Kurulumu 
# Projeyi indir
cd /opt
sudo git clone https://github.com/tulparstudyo/kilit.git tulpar-kilit
cd tulpar-kilit/backend

# Bağımlılıkları kur
npm install --production
5 Backend Yapılandırması

/opt/tulpar-kilit/backend/.env dosyasını oluşturun:

PORT=3000

# Güvenlik — rastgele uzun değerler kullanın
JWT_SECRET=RASTGELE_UZUN_BIR_ANAHTAR_BURAYA
OFFLINE_SECRET=RASTGELE_UZUN_BIR_ANAHTAR_BURAYA
ADMIN_SECRET=RASTGELE_UZUN_BIR_ADMIN_SIFRESI

# Veritabanı
DB_HOST=localhost
DB_USER=tulpar
DB_PASSWORD=GÜÇLÜ_BİR_ŞİFRE
DB_NAME=tulpar_kilit
DB_PORT=3306

# SMTP (e-posta gönderimi)
SMTP_HOST=smtp.example.com
SMTP_USER=kullanici@example.com
SMTP_PASS=smtp_sifreniz
SMTP_PORT=587
SMTP_SENDER=Tulpar Kilit<noreply@example.com>

# Yönetici bildirim e-postası
EMAIL_TO=admin@example.com

APP_VERSION=1.0.0
⚠️ Secret değerleri için terminalde openssl rand -hex 32 komutuyla rastgele anahtar üretebilirsiniz.
6 Systemd Servisi

Backend'in sunucu yeniden başlatıldığında otomatik çalışması için:

# Servis dosyası oluştur
sudo tee /etc/systemd/system/tulpar-backend.service > /dev/null << 'EOF'
[Unit]
Description=Tulpar Kilit Backend
After=network.target mysql.service

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/tulpar-kilit/backend
ExecStart=/usr/bin/node server.js
Restart=on-failure
RestartSec=5
Environment=NODE_ENV=production

[Install]
WantedBy=multi-user.target
EOF

# Servisi etkinleştir ve başlat
sudo systemctl daemon-reload
sudo systemctl enable tulpar-backend
sudo systemctl start tulpar-backend

# Durumu kontrol et
sudo systemctl status tulpar-backend
7 Nginx Reverse Proxy ve SSL

HTTPS ile dış dünyaya açmak için Nginx yapılandırması:

# Nginx site yapılandırması
sudo tee /etc/nginx/sites-available/tulpar-kilit > /dev/null << 'EOF'
server {
    listen 80;
    server_name kilit.sizinalanadiniz.com;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
EOF

# Siteyi etkinleştir
sudo ln -s /etc/nginx/sites-available/tulpar-kilit /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx

# SSL sertifikası al (Let's Encrypt)
sudo certbot --nginx -d kilit.sizinalanadiniz.com
⚠️ kilit.sizinalanadiniz.com yerine kendi alan adınızı yazın. DNS kaydının sunucunuzun IP adresine yönlendirilmiş olması gerekir.
8 CORS Ayarı

server.js dosyasındaki CORS origin değerini kendi alan adınızla güncelleyin:

// server.js içinde
app.use(cors({
  origin: ['https://kilit.sizinalanadiniz.com'],
  credentials: true
}));

Değişiklikten sonra servisi yeniden başlatın:

sudo systemctl restart tulpar-backend
9 Masaüstü İstemcilerini Bağlama

Masaüstü kurulumlarında (Aşama 1) API_URL değerini kendi sunucu adresinize yönlendirin:

# desktop/.env
API_URL=https://kilit.sizinalanadiniz.com
✅ Sunucu kurulumu tamamlandı. https://kilit.sizinalanadiniz.com adresinden web arayüzüne erişebilir, masaüstü istemcilerini bu adrese yönlendirebilirsiniz.
